# STRØNG·LOG — установка на хостинг Beget (чек-лист для исполнителя)

Это пошаговая инструкция «что нажимать». Программировать ничего не нужно —
только загрузить файлы и выполнить несколько действий в панели Beget.
Время: ~30–60 минут. Нужны: доступ в панель Beget и привязанный домен (или поддомен).

В архиве папка **`webroot/`** — это ровно то, что должно лежать в корне сайта.

Итоговая раскладка на сервере:

```
<корень сайта>/            (на Beget это обычно ~/<домен>/public_html)
  index.html               ← интерфейс
  assets/                  ← скрипты и стили
  .htaccess                ← перенаправление на https + маршрутизация
  api/                     ← серверная часть (PHP)
    config.local.php       ← СОЗДАЁТСЯ ВРУЧНУЮ на шаге 4 (пароли, в архиве его НЕТ)
  db/                      ← служебные файлы установки (запускаются один раз)
```

---

## Шаг 1. Создать базу данных MySQL

1. Панель Beget → раздел **«Базы данных» (MySQL)**.
2. Нажать **«Создать базу данных»**.
3. Записать и сохранить (понадобятся на шаге 4):
   - **имя базы** (напр. `login_stronglog`)
   - **имя пользователя** (часто совпадает с именем базы)
   - **пароль** базы
   - **хост** — обычно `localhost`

---

## Шаг 2. Загрузить файлы сайта

1. Панель Beget → **«Файловый менеджер»** (или подключиться по FTP).
2. Открыть корень нужного домена (папка **`public_html`** внутри папки домена).
3. Загрузить туда **всё содержимое папки `webroot/`** из архива
   (файлы `index.html`, `.htaccess` и папки `assets/`, `api/`, `db/`).
   - Важно: загружаем именно *содержимое* `webroot/`, а не саму папку `webroot`.
   - Файл `.htaccess` начинается с точки — убедиться, что он тоже загрузился
     (в файловом менеджере включить показ скрытых файлов, если их не видно).

---

## Шаг 3. Загрузить структуру базы и историю тренировок

Два способа на выбор — **A (проще, через панель)** или **B (через консоль)**.

### Вариант A — через phpMyAdmin (рекомендуется)

1. Панель Beget → **phpMyAdmin** → войти под пользователем базы из шага 1.
2. Слева выбрать свою базу.
3. Вкладка **«Импорт»** → выбрать файл **`db/schema.sql`** из архива → **«Вперёд»**.
   Это создаст пустые таблицы.
4. Историю (10 прошлых тренировок) удобнее залить на шаге 5 командой
   `php db/seed.php`. Если консоли (SSH) не будет — напишите нам,
   подготовим SQL-файл с историей для импорта тем же способом.

### Вариант B — через SSH (если есть доступ)

```bash
mysql -u ПОЛЬЗОВАТЕЛЬ -p ИМЯ_БАЗЫ < db/schema.sql
```

---

## Шаг 4. Прописать пароли (файл `api/config.local.php`)

Этого файла в архиве намеренно нет — он содержит секреты и создаётся на месте.

1. В файловом менеджере зайти в папку **`api/`**.
2. Скопировать файл **`config.local.example.php`** и переименовать копию в
   **`config.local.php`**.
3. Открыть `config.local.php` на редактирование и заполнить:

```php
<?php
return [
    'token'   => 'ВАШ_ДЛИННЫЙ_СЕКРЕТ',   // это пароль для входа в приложение
    'db_host' => 'localhost',
    'db_name' => 'имя_базы_из_шага_1',
    'db_user' => 'пользователь_из_шага_1',
    'db_pass' => 'пароль_базы_из_шага_1',
];
```

- **`token`** — это пароль, который будете вводить при входе в приложение.
  Сделайте его длинным и случайным. Пример готового значения (можно взять этот
  или сгенерировать свой):
  `8b345d772488a8bf3744e469fa6d5bbe1d9de4d18f407d78`
- Остальные 4 поля — данные базы из шага 1.

4. Сохранить файл.

---

## Шаг 5. Залить историю тренировок (если есть SSH)

Если делали импорт через phpMyAdmin (вариант A) и таблицы созданы:

```bash
cd <корень сайта>
php db/seed.php
```

Скрипт выведет 10 строк «✓ …» и «Засижено тренировок: 10». Повторный запуск
безопасен (дубликатов не создаёт). Если SSH нет — пропустите, историю зальём отдельно.

---

## Шаг 6. Включить SSL (https)

1. Панель Beget → **«SSL-сертификаты»** → выбрать домен → выпустить
   **бесплатный сертификат Let's Encrypt**.
2. Подождать активацию (обычно несколько минут).
   Принудительное перенаправление на https уже встроено в `.htaccess` —
   отдельно настраивать не нужно.

---

## Шаг 7. Проверка

1. Открыть домен в браузере — должен появиться экран входа STRØNG·LOG,
   адрес — на `https://`.
2. Ввести **token** из шага 4 → должно открыться приложение.
3. На вкладке **«Тренировки» → «История»** должны быть видны 10 прошлых
   тренировок (если выполняли шаг 5).
4. Записать тестовую тренировку, открыть сайт **с телефона**, войти тем же
   токеном — запись должна быть на месте (это и есть синхронизация между
   устройствами).

Готово. 🎉

---

## Если что-то не так

- **Белый экран / 500** → чаще всего не заполнен или с ошибкой `api/config.local.php`
  (шаг 4), либо неверные данные базы.
- **«Неверный токен»** → token в приложении не совпадает с `token` в
  `config.local.php`.
- **История не появилась** → не выполнен шаг 5 (`php db/seed.php`) или таблицы
  не созданы (шаг 3).
- **Сайт открывается по http, не по https** → ещё не выпущен SSL (шаг 6).

При любой заминке пришлите скриншот ошибки — разберёмся.
